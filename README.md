# Time ROI Hub

> 「その買い物、本当に時間的に得ですか？」を一瞬で判断する ROI 計算アプリ

---

## サービス概要

**Time ROI Hub** は、家電・サブスク・生活改善サービスなどの購入に対して、「どれくらい時間を節約できるか」をもとに投資回収期間（ROI）を算出し、購入判断を支援する Web アプリケーションです。

レビューサイトや比較サイトでは「良い商品か」は分かっても、「自分にとって時間的に得か」は分かりません。本サービスは「時間価値」に特化することで、感覚ではなく“回収期間”で生活の意思決定をサポートします。

### 解決する課題

| 課題 | 解決策 |
|------|--------|
| 買うべきかどうか判断できない | ROI（投資回収期間）を自動計算 |
| 時短効果が実感しづらい | 月間・年間の削減時間を可視化 |
| 比較サイトは情報量が多すぎる | 入力3項目だけの即断 UI |
| 感覚で購入して後悔してしまう | 時間価値ベースで合理的に判断 |

---

## 開発背景

日常生活の中で、家電やサブスクを購入する際に「本当にこれを買う価値があるのか？」と迷うことが何度もありました。

しかし、既存の比較サイトやレビューサイトでは「スペック」や「口コミ」は分かっても、「その商品が人生の時間をどれくらい増やしてくれるか」を判断することはできませんでした。

また、「時間はお金より重要」という考え方は広く知られている一方で、実際に生活改善を“時間投資”として数値化できるサービスが少ないことに違和感を感じました。

そこで、「生活の意思決定を時間ROIで支援するツール」を作りたいと考え、本サービスを開発することにしました。

## 機能一覧

### MVP リリース

- **ROI 計算機能**：価格・時短・時給を入力して投資回収期間を計算
- **回収期間表示**：何ヶ月で元が取れるかを可視化
- **購入判断ラベル**：「買い」「保留」「微妙」を自動表示
- **家電プリセット機能**：食洗機・ロボット掃除機などの時短データを即利用可能
- **レスポンシブ UI**：スマホ前提のシンプル設計

### 本リリース（実装予定）

- 商品比較機能（A vs B）
- ROI 履歴保存機能
- カテゴリ別プリセット一覧
- SEO 用個別ページ生成
- アフィリエイト導線表
- 快適度スコア表示
- localStorage による時給保存

## 技術スタック

### フロントエンド

| 技術 | 採用理由 |
|------|----------|
| **Next.js 15（App Router）** | SEO と高速表示を両立しつつ、将来的な拡張性を持たせるため |
| **React 19** | コンポーネント単位で ROI カードや比較 UI を再利用しやすくするため |
| **TypeScript** | ROI データやプリセットデータの型安全性を保つため |
| **Tailwind CSS v4** | モバイルファーストな UI を高速に構築するため |
| **shadcn/ui** | 高いカスタマイズ性でシンプルかつモダンな UI を実現するため |
| **Radix UI** | アクセシブルな UI プリミティブとして使用 |
| **class-variance-authority / clsx / tailwind-merge** | 条件付きスタイル管理と Tailwind の競合解消のため |
| **lucide-react** | 軽量で統一感のあるアイコンを実装するため |
| **Geist フォント** | モダンで視認性の高い UI を実現するため |

### バックエンド

| 技術 | 採用理由 |
|------|----------|
| **バックエンドなし（MVP）** | ROI 計算のみで完結するため、開発速度を優先してフロント完結構成を採用 |
| **localStorage** | 時給や履歴をブラウザに保存するため |
| **将来的に Supabase / Rails API を検討** | ユーザー保存機能や分析機能追加時にスケール可能な構成にするため |

### 開発・テスト

| 技術 | 採用理由 |
|------|----------|
| **Vitest** | 高速なフロントエンドテスト実行のため |
| **Testing Library（React）** | ユーザー操作ベースで UI をテストするため |
| **ESLint** | コード品質を保つため |
| **Prettier** | コードフォーマットを統一するため |

### インフラ

| 技術 | 採用理由 |
|------|----------|
| **Vercel** | Next.js との親和性が高く、デプロイが容易なため |

## ローカル環境での起動方法

### 前提条件

- Node.js 20 以上
- npm または pnpm

### セットアップ

```bash
git clone https://github.com/nagi558/Time-ROI-Hub
cd time-roi-hub
docker-compose up
```

### 開発サーバー起動

```bash
npm run dev
```

ブラウザで以下へアクセス：

```bash
http://localhost:3000
```

### ビルド

```bash
npm run build
```

## こだわりポイント

- **意思決定特化 UI**：「調べる」ではなく「決める」ことにフォーカス
- **入力最小設計**：価格・時短・時給の3項目だけで即結果表示
- **スマホファースト**：親指操作のみで完結する UX
- **プリセット設計**：家電・サブスクなどをワンタップで比較可能
- **時間価値の可視化**：単なる価格比較ではなく、“人生の時間”で判断できる設計
